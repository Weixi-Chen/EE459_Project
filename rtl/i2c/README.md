# `rtl/i2c/` — I2C 主机与传感器协议

本目录放 **共享 I2C 总线** 上的 **单主机** 实现，以及 **BH1750（光照）**、**DHT20（温湿度）** 的寄存器级状态机。

## 建议在此目录编写的文件

| 文件（建议名） | 作用 |
|----------------|------|
| `i2c_master.v` | START、发送/接收字节、ACK 采样、STOP；在 50 MHz 下用 `i2c_tick_en` 分频满足 ~100 kHz（或更低）I2C 时序。 |
| `bh1750_ctrl.v` | BH1750：配置测量模式、触发读数；输出原始计数值或换算后的 lux（换算也可放在顶层组合逻辑）。 |
| `dht20_ctrl.v` | DHT20：严格按 **Adafruit 5183 / 芯片数据手册** 写上电等待、初始化、测量命令与读回解析（地址与命令以手册为准，勿硬抄其它项目）。 |

## 硬件注意

- **SDA** 在 FPGA 侧通常实现为 **开漏行为**（`inout` + 高阻释放总线），**SCL** 同理若需时钟延展；与课程模板一致即可。
- BH1750 与 DHT20 **共用 SDA/SCL**；同一时刻只允许 **一个** 高层控制器占用 `i2c_master`（由 `app/sensor_arbiter` 调度）。

## 与顶层的边界

- 本目录 **不** 决定「何时刷新界面」；只响应仲裁器的 `start`/`addr`/`rw`/数据缓冲并完成 `busy`/`done`/`error`。
